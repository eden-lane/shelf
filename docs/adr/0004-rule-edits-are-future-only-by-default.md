# Rule edits are future-only by default

Editing import rules affects future saved items only. Existing saved items are not automatically reorganized when rules change; users need a separate rule reapply operation to evaluate current rules against already imported provider items.

## Consequences

Rule edits are safe and predictable because they do not rewrite a curated library. A future rule reapply feature should be explicit, bulk-oriented, and confirmable before moving folders or adding tags to existing saved items.
